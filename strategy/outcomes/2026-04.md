## Security Incidents

### [SECURITY] Company Audit — 2026-04-28

**Finding:** CISO Agent flagged commit `5a9ac86` in `audiotext-app` for containing the sensitive keyword `token` in its commit message.

**Investigation:** The commit (`fix: resolve 4 recurring CI/CD workflow failures`) was a false positive. The squash-merged commit body included GitHub Actions workflow YAML referencing `${{ secrets.GITHUB_TOKEN }}` — a standard CI/CD pattern, not a leaked credential. No actual secrets were exposed.

**Resolution:**
- Decision → audiotext-app #99 and #107 investigated and closed
- Deploy → audiotext-app commit `e091469`: PR bodies are now sanitized before squash-merge to strip GitHub Actions expressions, preventing future CISO false positives
- Result → HQ CISO audit `safeCommitPatterns` expanded to recognize CI/CD workflow token patterns (`${{ secrets.GITHUB_TOKEN }}`, `github.token`, `github-token:`, `persist-credentials`, etc.)

**Preventive action:** Both the product repo (sanitize before merge) and the HQ audit workflow (broader safe patterns) were hardened to reduce recurrence.
