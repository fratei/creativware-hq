## Security Incidents

### [SECURITY] Company Audit — 2026-04-29

**Finding:** CISO Agent (HQ issue #38) flagged commit `5a9ac86` in `audiotext-app` for containing the sensitive keyword `token`.

**Investigation:** Duplicate false positive — the same commit was already reviewed and closed as `completed` in HQ issues #28 and #35 (2026-04-28 audits). The commit body references `${{ secrets.GITHUB_TOKEN }}` in CI/CD YAML; no actual credentials were exposed.

**Root cause:** The 7-day rolling audit window causes already-reviewed commits to be re-flagged on subsequent days if they remain in the scan range. The `safeCommitPatterns` update from 2026-04-28 had not yet been merged when the 2026-04-29T02:27 audit ran.

**Resolution:**
- Decision → HQ issue #38 investigated and confirmed false positive (no credentials to rotate, no history purge needed)
- Deploy → HQ `agent-reconciler.yml`: CISO audit now builds a set of already-reviewed commit SHAs from recently closed CISO security issues and skips them, preventing duplicate alerts across the rolling window
- Result → Commit `5a9ac86` will not trigger further `⚠️` findings; remaining days in the rolling window will produce no alert for this SHA

**Preventive action:** CISO audit dedup logic added: before scanning, the audit loads commit SHAs mentioned in closed `[SECURITY]` issues (last 2× rolling window) and excludes them from the current run.
